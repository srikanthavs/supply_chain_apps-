3. Screen Inventory
Screen 1 — Skill World Map (Home)
Layout: Full-bleed cosmic canvas (1200×800) with starfield + radial gradient tinted by current mode. Pulsing MathWorld sun centered at (600, 400). Filtered orbital rings (1 ring if ≤8 planets, 2 rings if more) with planets evenly spaced. HUD top-left/right (greeting chip, streak, stars, Archie XP, parent lock). Mode banner top-left with counts. Archie + speech bubble bottom-left. Mode bar bottom-center.

Key interactions: Tap any of 4 mode tabs → background tint, planet set, Archie copy, and orbital rings transition smoothly. Tap planet → opens stub popup with mastery, prereqs, and 4 mode launch CTAs. Tap parent lock → PIN sheet. Hover/tap micro-bounce on every interactive element.

Connects to: Skill Detail (planet tap), Parent Dashboard (lock tap), and conceptually feeds all 4 learning modes via the popup CTAs.

Screen 2 — Skill Detail
Layout: Two-column 1280×820 canvas. Left column (440px): class chip → big planet hero with dust ring, moon, floating stars badge → skill title and subtitle → Archie pointing with contextual speech bubble. Right column (flex): Mastery card with concentric ring chart + description + 3 layer chips (Concrete/Pictorial/Abstract) → 2×2 grid of Mode Launch Cards (Recommended pip on Practice, Speed locked because Abstract isn't mastered) → 3-up footer row (Built-on prereqs ✓, Recent visits log, "Master to unlock" dashed yellow card).

Key interactions: Back to Map button (top-left), mode card tap launches that mode for this skill, locked Speed card stays disabled with explanation, suggest-activity buttons are touch targets parents could tap.

Connects to: Back to Skill World Map; forward to all 4 learning modes (Explore/Practice/Challenge/Speed) carrying the selected skill context.

Screen 11 — Parent Dashboard
Layout: Scrollable single page behind PIN gate (1234). Sticky header with Archie L3 icon, child name, and 4 metric chips (level/XP, stars, streak, last session) plus "🔒 Lock" button. Body is a 1.45:1 two-column grid: left column = Skill Progress Overview (4-col grid of all 23 skills grouped by Level 1/2/3 with mastered/in-progress/locked variants) and Last 7 Sessions table (date, skill, mode pill, accuracy bar, duration). Right column = Top 3 Weak Areas (numbered cards with error-type pill, attempts, plain-English advice, suggest-activity CTA) and Weekly Activity bar chart (purple gradient bars, ★ best-day pip, gridlines, auto-summary sentence).

Key interactions: PIN entry with shake-on-error and auto-submit at 4 digits. Lock button returns to PIN screen. Suggest-activity buttons open per-skill coaching prompts (not built — Claude Code to wire). Hover lift on cards.

Connects to: Reached via discreet 🔒 lock icon on Skill World Map. "Lock" button returns parent to MathWorld home.

4. What Was NOT Built — Remaining Screens for Claude Code
All remaining screens must follow the design system above. Each screen must show Archie somewhere (mood appropriate to context) and use the mode color zone indicated.

Screen 3 — Explore Mode (Purple zone)
Active learning canvas with draggable manipulatives (blocks, coins, number lines, shapes). No score, no timer, no right/wrong. Archie narrates a small story in a persistent speech bubble. Top bar: skill name, exit, mode pill. Workspace: tappable/draggable objects with snap-to-grid feel. Bottom: "next idea" button to advance Archie's narration.

Screen 4 — Practice Mode: Concrete Layer (Blue zone)
Visual problem with manipulatives the child arranges. e.g. "Show 23 + 14 with blocks." Drop zones for tens/ones, "Check my answer" button. Layer progress bar at top showing Concrete → Pictorial → Abstract.

Screen 5 — Practice Mode: Abstract + Reasoning (Blue zone)
Equation form (e.g. 23 + 14 = ?) with 4 large answer tiles. After correct answer, second card slides in: "How did you know?" with 2–3 reasoning choice buttons. Archie reacts to choice.

Screen 6 — Wrong Answer + Hint (Blue zone)
Gentle wobble on the answer, never harsh red. Archie animates to "thinking" pose. Hint card appears, classified by error type:

Conceptual → different visual representation
Calculation → simplified numbers, same concept
Careless → "slow down, double-check" prompt with re-presentation of question
"Try again" button. No penalty until 3 wrong.

Screen 7 — Correct Answer Celebration (mid-session)
Star burst / confetti from the answer location. Archie cheers (wand up, sparkle pulse). +XP toast slides in. Auto-advance after ~1.2s. Inline, non-blocking.

Screen 8 — Skill Completed / Unlocked (full-screen)
Full-screen takeover. Archie levels up with transform animation (new color in robe trim, new wand glow, "LV 5" hat band). Big "Planet Mastered!" banner. Gold ring grows around planet. Show stars earned (e.g. 9/9), skills now unlocked next, "Back to Map" CTA. Optional confetti.

Screen 9 — Challenge Mode (Orange zone)
Big real-world word problem in a card (no scaffolding). Archie wears a thinking cap (variant of Archie SVG with cap added). Hint button costs 1 star. Single text answer or short multiple-choice. "Deep Thinker" badge teased at corner if answered without hint.

Screen 10 — Speed Mode with Timer (Yellow zone)
Duration picker first (60s / 90s pills). Then: large countdown ring, fact question center, 4 large answer tiles. Streak counter with Archie running faster as correct answers stack (Archie variant with motion lines). End screen: accuracy %, total answered, "Run again" / "Back to Map".

Implementation Notes for Claude Code
Use React + Tailwind. Framer Motion is preferred for animations; the existing screens use a thin shim because of UMD loading — feel free to replace with the real package in a proper build.
Fonts: load Fredoka One + Nunito (400/600/700/800/900) from Google Fonts.
Tablet-first at 1024×768 minimum, scale up cleanly to desktop. Each screen wraps in a <Stage> that auto-scales a fixed-size canvas to viewport.
No dark backgrounds on any child screen. Backgrounds are tinted radial gradients keyed off the mode color.
Archie appears on every child screen; pose matches context (point, celebrate, think with cap, run, level-up).
Errors are warm, never harsh — coral wobble, not red flash. Wrong answers go straight into the hint flow.
All interactive elements ≥ 60px on child screens.
State persistence is your job — these prototypes are presentational only.
